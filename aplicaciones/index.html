    DPKG

    Los archivos .deb, son ejecutables en Ubuntu(Debian) y permiten la instalación del software elegido. Su instalación es realizada mediante el comando dpkg

    dpkg [ opciones ] aplicación

    Opciones

    Comando Acción
    dpkg -I agustinPrg.deb Nos muestra información del paquete “agustinPrg”
    dpkg -i agustinPrg.deb Instala el paquete “agustinPrg”
    dpkg -r agustinPrg Desinstala el paquete “agustinPrg”
    dpkg -S /usr/bin/agustinPrg Nos indica el propietario del fichero "agustinPrg"
    dpkg -L agustinPrg Muestra el listado de los ficheros que pertenecen al paquete "agustinPrg"
    dpkg -s agustinPrg Muestra el estado del paquete "agustinPrg"
    dpkg -c agustinPrg.deb Muestra los ficheros que contiene el paquete deb ANTES de instalarlo

    Repositorios

    Apt es una herramienta de administración que encontraremos en todos los sistemas que están basados en Debian, es el que se encarga de buscar los paquetes que queremos instalar en los repositorios. Un ejemplo de uso de apt sería:

    apt [ opciones ] aplicación

    Opciones

    Comando Acción
    apt install nombre_paquete instalación
    apt install --reinstall nombre_paquete re-instalación
    apt remove nombre_paquete Desinstalación
    apt update Actualización del catálogo
    apt check Comprobar que no hay dependencias
    incumplidas
    apt-cache search patron Búsqueda en el catálogo
    apt-cache show nombre_paquete Mostrar información de un paquete
    apt-cache policy nombre_paquete Mostrar información acerca de las versiones
    apt full-upgrade Actualizar el sistema (completa)
    apt upgrade Actualizar el sistema (cobarde)
    apt autoremove Limpiar el sistema de paquetes innecesarios
    apt clean Limpiar el catalogo

    Compilando código

    Generalmente las aplicaciones en código fuente vienen en este formato .tar . Será necesario su descompresión y desempaquetamiento para pasar al proceso de linkar o enlazar la librerías necesarias para el funcionamiento de la aplicación y la compilación de la aplicación en ejecutable.

    make && make install

    tar

    El comando TAR es usado empaquetar (o comprimir si es indicado) un archivo o directorio:

    Opciones

    Comando Acción
    c Crea un paquete
    t Comprueba los archivos almacenador en paquete
    x Extrae los archivos que contiene el paquete
    v Muestra el proceso de empaquetamiento
    p Conserva los permisos de los ficheros
    r Añade archivos al paquete
    f Especifica el nombre del paquete
    z Comprime o descomprime

    gzip

    La orden Gzip es un comando que permite comprimir. El nombre del archivo del archivo resultante se identifica con la extensión .gz y debe de ser descomprimido antes de ser usado

    Opciones

        gzip agustin.doc                #Comprimir

    gzip -d agustin.doc.gz          # Descomprimir, opción a
    gunzip agustin.doc.gz           # Descomprimir, opción b

    Bzip2

    Permite comprimir archivos. Comprime más que gzip (el algoritmo utilizado es más completo). La extensión que crea es .bz2

    Opciones

        bgzip agustin.doc               # Comprimir

    bgzip -d agustin.doc.bz2        # Descomprimir, opción a
    gunzip agustin.doc.bz2          # Descomprimir, opción b
